# DSPy ドキュメントのカスタマイズ方法


本ウェブサイトは、MKDocs用のMaterial UI風テーマである[Material for MKDocs](https://squidfunk.github.io/mkdocs-material/)を使用して構築されています。

## ローカル環境でのドキュメントビルド

ドキュメントをローカル環境でビルドおよびテストするには：

1. `docs` ディレクトリに移動してください：
   ```bash
   cd docs
   ```

2. 必要な依存関係をインストールします：
   ```bash
   pip install -r requirements.txt
   ```

3. `docs/` ディレクトリ内で、以下のコマンドを実行して API ドキュメントを生成し、インデックスを作成してください：
   ```bash
   python scripts/generate_api_docs.py
   python scripts/generate_api_summary.py
   ```

4.（オプション）MacOS環境では、サイト構築に必要なライブラリのインストールも必要となる場合があります
   ```bash
   brew install cairo freetype libffi libjpeg libpng zlib
   export DYLD_FALLBACK_LIBRARY_PATH=/opt/homebrew/lib
   ```

5. ビルドコマンドを実行する：
   ```bash
   mkdocs build
   ```

これにより、`site`ディレクトリ内にドキュメントサイトの静的ビルドが生成されます。生成されたディレクトリを以下の方法で公開することで、ローカル環境でサイトを閲覧可能です：

```bash
mkdocs serve
```

ビルドが失敗しているのを確認した場合は、マージを行う前に必ず修正してください。

## 継続的インテグレーション（CI）ビルドチェックについて

ドキュメントの変更をマージする前にビルドが正常に完了することを保証するため、CIパイプラインに自動化されたビルドチェックを導入しています。これらのチェックでは以下を行います：

1. `mkdocs build`コマンドを実行
2. ビルドがエラーなく完了することを確認
3. 開発プロセスの早い段階で潜在的な問題を発見可能にする

CIビルドチェックが失敗した場合は、変更内容を再確認し、ローカル環境でドキュメントが正しくビルドできることを確認した上で更新をプッシュしてください。

## `docs`フォルダへの貢献について

本ガイドは、`dspy/docs`フォルダ内のドキュメントに変更を加えようとする貢献者向けのものです。

1. **最新のウェブサイトバージョンを取得**：dspyリポジトリをクローンして、ライブドキュメントサイトの最新バージョンを取得してください。現在のドキュメントは`dspy/docs`フォルダに格納されています。

2. **新規ブランチで変更をプッシュ**：既存のドキュメントの追加や編集を自由に行い、変更内容に対するプルリクエストを作成してください。PRがレビューされ承認されると、変更はメインブランチにマージ可能な状態になります。

3. **ウェブサイトの更新**：変更がメインブランチにマージされると、通常5～15分程度でライブウェブサイトに反映されます。

## LLMs.txtファイルについて

ビルドプロセスでは、[mkdocs-llmstxt](https://github.com/pawamoy/mkdocs-llmstxt)を使用してLLM向けの`/llms.txt`ファイルを生成します。`mkdocs.yml`ファイル内の`llmstxt`プラグインセクションで設定を行ってください。
